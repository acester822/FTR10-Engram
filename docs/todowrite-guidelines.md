# todowrite Usage Guidelines

## When to Use `todowrite`:
- Tasks with **3+ steps** — track progress explicitly so you don't lose your place or hallucinate completion.
- Complex, multi-step tasks — helps organize and manage the workflow.
- After receiving new instructions — immediately capture user requirements as todos.
- After completing a task — mark it complete and add any follow-up tasks.

## When NOT to Use `todowrite`:
- Single, straightforward tasks (1-2 steps)
- Trivial tasks that don't require tracking
- Purely conversational or informational requests

## Best Practices:
1. **Track progress explicitly** — mark items as in_progress when starting, completed when done
2. **Keep only ONE item in_progress at a time** — complete existing tasks before starting new ones
3. **Update the list after each step** — cancel items that are no longer needed
4. **Break complex tasks into smaller, actionable items** with clear descriptions
